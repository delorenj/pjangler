# pjangler — Component Inventory

> Generated by BMAD `document-project` (deep scan) · 2026-07-13
> Catalog of the tool's addressable "components": recipes, commands, parity rules, and MCP tools. For a CLI this replaces the UI-component inventory.

## Recipes (`RECIPE_REGISTRY` — `src/utils/registry.ts`)

A recipe bootstraps a whole subsystem by composing Commands. Run with `pj add <name>` or `pj recipe run <name>`.

| Recipe | Description | Ingredient commands |
| --- | --- | --- |
| `mise` | Mise task runner + environment setup | `AddMiseToml`, `AddDotenv`, `AddMiseTasksStructure`, `AddMiseBaseToml`, `AddMiseBaseScript`, `AddMiseCodegraphScript` |
| `docker` | Docker containerization setup | `AddDockerfile`, `AddDockerCompose`, `AddDockerignore` |
| `node` | Node.js project template | `AddPackageJson`, `AddReadme`, `AddSrcDirectory` (`NodeCommands.ts`) |
| `hermes-agent` | Add a Hermes agent role (copier + BotFather + CF email + submodule) | `EnsureTemplateConfig`, `PromptForAgentConfig`, `RunCopierTemplate`, `UntrackHermesRuntimes`, `WireTelegram`, `WireEmail`, `PrintHermesSummary` |
| `agent-hooks` | Retrofit the project-scoped agent-hooks + skill fan-out layer | `CopyAgentHooksTree`, `WireMiseAgentHooks` |

> Note: `RECIPE_REGISTRY` lists `hermes-agent` commands as the 7-step chain minus `UntrackHermesRuntimes`, but `HermesAgentRecipe.ts` includes it as step 4. The recipe class is authoritative for execution.

## Parity rules (`RULES` — `src/parity/index.ts`)

Deterministic, idempotent audit/migrate rules enforcing the 33god project standard. `pj audit` reports status per rule; `pj migrate <rule-id>` (or `--all`) applies fixes. Audit status ∈ `pass | fail | warn | skip`; migrate status ∈ `applied | noop | blocked | skipped`.

| Rule id | Title | Fixable | Enforces |
| --- | --- | --- | --- |
| `sot.project-json` | Canonical `.project.json` | ✅ | Required keys, correct `repo_path`, discovered agents, `ticket_provider` block (no legacy `board_url`), `automation.reconcile`; flags stray `.plane.json` |
| `mise.config-root` | mise config_root + AGENTS link hooks | ✅ | `mise.toml` present with AGENTS-link enter hook + `op inject` + `_.path` entries (space-safe `[[hooks.enter]]`) |
| `mise.versioning` | mise versioning tasks | ✅ | Managed `mise-versioning` task block + `.mise/scripts/versioning.sh` |
| `sot.agent-symlinks` | AGENTS/CLAUDE/GEMINI symlink contract | ✅ | `CLAUDE.md` & `GEMINI.md` are symlinks → `AGENTS.md` (skips if `AGENTS.md` absent) |
| `secrets.env-op` | `.env.op` / op inject secrets | ✅ | 1Password `op inject` secrets wiring present |
| `provenance.copier` | copier provenance | ✅ | copier answers/provenance markers present |
| `bmad.scaffold` | BMAD scaffold parity | ✅ | BMAD `bmm,bmb,cis` installed for the full ~48-tool matrix |
| `bmad.version` | BMAD version parity | (check) | Installed BMAD matches the `next` channel (npm dist-tags cached <1h; `skip` when offline) |
| `hermes.pm-scaffold` | Hermes PM scaffold parity | ✅ | `agents/hermes/pm` role scaffold complete (skips if no PM role) |
| `hermes.untracked-runtimes` | Hermes untracked runtimes | ✅ | Runtime submodules untracked + gitignored, not committed |
| `systemd.sentinel` | Hermes systemd/sentinel units | ❌ | `--user` gateway/consumer/heartbeat units enabled + active (warn/skip when systemd unavailable) |

Get the live list at runtime: `pj audit --json` or MCP `pjangler_list_parity_rules`.

## MCP tools (`src/mcp-server.ts`)

Server `pjangler-mcp`. All reads/plans are safe; writes require explicit `apply`/`force`, network/cloud require `local: false`. Responses embed `@33god-projects` skill guidance.

| Tool | Purpose | Safety default |
| --- | --- | --- |
| `pjangler_list_capabilities` | Recipes, commands, parity rules, recommended workflows, skill synergy | read-only |
| `pjangler_list_parity_rules` | Parity rule ids + guidance | read-only |
| `pjangler_audit_project` | Parity audit with summary counts + next actions | read-only |
| `pjangler_migrate_project` | Run one rule or `all` | `dryRun: true` |
| `pjangler_bootstrap_33god_project` | Create a new CommonProject repo (+ optional local Hermes) | `dryRun: true`, `local: true`, `skipPlane: true` |
| `pjangler_project_init` | Plan/apply registry-backed init | `apply: false` |
| `pjangler_project_list` | List registry projects | read-only |
| `pjangler_project_show` | Show one project by slug | read-only |
| `pjangler_describe_recipe` | Recipe metadata | read-only |
| `pjangler_run_recipe` | Run any recipe against a target dir | applies (honor `dryRun`) |
| `pjangler_deploy_hermes_agent` | Provision a Hermes role (pm/dev/review/ops/qa) | `local: true` (skips runtime repo/Plane/Bloodbank/systemd) |
