## Entry points

- **CLI:** `src/index.ts` → `program.parse()` (commander). Binaries `pjangler` / `pj` map to `dist/index.js`.
- **MCP:** `src/mcp-server.ts` → `server.connect(new StdioServerTransport())`. Binary `pjangler-mcp` maps to `dist/mcp-server.js`.
- **Package root resolution:** `resolvePjanglerRoot()` (in `mcp-server.ts`, `project/index.ts`, `parity/index.ts`) walks up from the module dir until it finds `package.json` + `templates/commonproject/copier.yml`, so the tool works both from `src/` (dev) and bundled `dist/` (install).

## Critical directories (by weight)

| Directory | Why it matters |
| --- | --- |
| `src/parity/` | The reconciler; most business logic and the trickiest code (TOML surgery). |
| `src/project/` | Project identity, registry, and the bootstrap plan/apply engine. |
| `src/commands/`, `src/recipes/` | The extensible scaffolding surface; add subsystems here. |
| `templates/` (submodules) | What a bootstrapped project/agent actually gets. |
| `tests/` | The de-facto correctness contract (no unit framework — regression suites). |

## Notable exclusions (`tsconfig.json`)

`agents/`, `dist/`, `node_modules/`, `plugins/`, `templates/`, `worktrees/` are excluded from typechecking. Only `src/**/*.ts` is compiled.
