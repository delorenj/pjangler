# pjangler ‚Äî Deployment & Release Guide

> Generated by BMAD `document-project` (deep scan) ¬∑ 2026-07-13
> "Deployment" for pjangler means **publishing the npm package**. There is no server to deploy ‚Äî the artifact is the CLI/MCP binary set installed on developer machines.

## Distribution model

`@delorenj/pjangler` is published to npm and installed globally (or run via `npx`). Binaries:

| Binary | Maps to | Purpose |
| --- | --- | --- |
| `pjangler`, `pj` | `dist/index.js` | CLI |
| `pjangler-mcp` | `dist/mcp-server.js` | MCP stdio server |

Per prior packaging incidents (PJAN-2): this is **npm-only**, and the published tarball had gitignore/packlist landmines in the past ‚Äî verify the tarball contents before publishing.

## What ships (`package.json` `files` allowlist)

Only these are included in the npm tarball:

```
dist/                         # bundled CLI + MCP server
templates/                    # copier templates (commonproject, hermes-agent)
.mise/scripts/versioning.sh   # shipped so target repos get the versioning workflow
.mise/scripts/link-agentfiles.sh
```

The parity engine reads these shipped scripts from the package root at runtime (`templateScript()`), so the `files` allowlist is load-bearing ‚Äî dropping a script silently breaks the corresponding parity rule.

## Build & publish flow

```bash
npm run build          # esbuild ‚Üí dist/index.js, dist/mcp-server.js (also runs via prepublishOnly)
npm run typecheck      # tsc --noEmit
npm test               # 4 regression suites
# bump version through the managed workflow (keeps package.json + git tag in parity):
mise run version:bump  # or :bump-minor / :bump-major
npm publish            # prepublishOnly re-runs build
```

`prepublishOnly: npm run build` guarantees `dist/` is fresh in the tarball. `PJANGLER_VERSION` is read from `package.json` at runtime, so the version reported by `pj --version` and the MCP `serverInfo` always matches the published `package.json` ‚Äî no separate bump needed in source.

### Pre-publish checklist

1. `npm run typecheck` clean.
2. `npm test` green (all four suites).
3. `mise run version:check` ‚Äî versioned files in parity.
4. Inspect the tarball: `npm pack --dry-run` ‚Üí confirm `dist/`, `templates/`, and both `.mise/scripts/*.sh` are present (PJAN-2 guard).
5. Confirm `templates/` submodules are populated (a bare submodule ships empty templates).

## CI

`.github/workflows/` contains Claude Code automation, not a build/test/publish pipeline:

- `claude.yml` ‚Äî Claude Code interactive workflow
- `claude-code-review.yml` ‚Äî automated PR code review

There is **no GitHub Actions test/publish job** at time of scan ‚Äî build, test, and publish are run locally via the commands above. (Adding a CI test gate that runs `npm run typecheck && npm test` on PRs would be a reasonable hardening step.)

## Runtime dependencies at the install site

- **Node ‚â• 20** (ESM).
- **`copier`** (Python) on `PATH` for any scaffolding/apply operation ‚Äî `uv tool install copier`. Missing copier is handled gracefully (surfaced as an actionable error, not a crash) but blocks the `copier.copy.commonproject` action.
- **`git`** for repo-root detection and submodule operations.
- Optional: **`op`** (1Password CLI) for `.env.op` injection; **`systemctl --user`** + **`mise`** on machines that run provisioned Hermes agents.

## Consuming pjangler from an agent (MCP)

Register `pjangler-mcp` as an MCP server (stdio). Agents should start with `pjangler_list_capabilities`, then use the audit ‚Üí migrate workflow. All tools default to safe (`dryRun`/`local`), so an agent cannot create cloud resources or overwrite files without explicit opt-out.
