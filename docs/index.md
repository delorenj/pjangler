## Project overview

- **Type:** Monolith — single npm package (CLI + MCP server)
- **Package:** `@delorenj/pjangler` v1.2.17
- **Primary language:** TypeScript (strict, ESM, Node ≥ 20)
- **Architecture:** Layered CLI with a command/recipe composition core, a plan/apply execution model, and a rule-based idempotent parity reconciler
- **Purpose:** Bootstrap and maintain 33GOD / DeLoNET projects — create from the CommonProject template, keep repos in parity with the 33god standard, provision Hermes PM agents, scaffold subsystems

## Quick reference

- **Tech stack:** commander (CLI) · @clack/prompts (TUI) · @modelcontextprotocol/sdk (MCP) · zod · yaml · esbuild · external `copier` (Python) · mise
- **Entry points:** `src/index.ts` (CLI: `pjangler` / `pj`), `src/mcp-server.ts` (MCP: `pjangler-mcp`)
- **Core subsystems:** `src/commands` + `src/recipes` (scaffolding) · `src/project` (registry + bootstrap) · `src/parity` (audit/migrate)
- **Central state:** `~/.config/pjangler/projects.yaml` (project registry)
- **Board:** Plane `pjangler` (PJAN) — binding in `.project.json`

## Getting started

```bash
mise trust && npm install && mise run setup   # setup
npm run build                                  # bundle CLI + MCP to dist/
npm run typecheck && npm test                  # gates (tsc --noEmit + 4 regression suites)
node dist/index.js --help                      # explore the CLI
```

To work on the tool: read [Architecture](./architecture.md) → [Development Guide](./development-guide.md), and consult the **`project-jangler`** skill for the Command/Recipe authoring workflow. All code changes require a Plane ticket on board **PJAN** (see AGENTS.md).
